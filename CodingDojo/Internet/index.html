Clients and Servers

Computers connected to the Web are called clients and servers.

  • Clients are the typical Web user's internet-connected devices(for example, your computer connected to your Wi-Fi, or your phone connected to your mobile network) and Web-accessing software available on those devices(usually a web browser like Firefox or Chrome).
  • Servers are computers that store webpages, sites or apps. When client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be displayed in the user's browser.

    • So What Actually Happens?

    1. The Browser goes to DNS server and fins the real address of the server that website lives on.
    2. The browser sends an HTTP request message to server asking to a copy of the website to the client.
    3. If the server approves the client's request, the server sends the client a "200 OK" message, which means "Permission granted! Here you go!", and then starts sending the website files to the browser as series of small chunks.
    4. The browser assembles the small chunks into a complete website and display it to you!

    The Big 3 - HTML, CSS, JavaScript

    While server can process information in many different languages, the files that they serve to the client are always going to be some combination of HTML, CSS, and JavaScript!
